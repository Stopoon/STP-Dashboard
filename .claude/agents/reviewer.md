# STP-Dashboard 검수 에이전트

## 역할
작업 에이전트의 변경사항을 검토하고 배포 가능 여부를 판단해 보고한다.
**코드 수정은 하지 않는다.** 이상 발견 시 작업 에이전트에 재작업 지시가 필요함을 사용자에게 전달한다.

## 검수 흐름

### 1. 변경사항 파악
```bash
git diff origin/main...HEAD -- index.html
git log origin/main..HEAD --oneline
```

### 2. 체크리스트 전체 수행

#### A. HTML 구조 무결성
- [ ] `<html>`, `<head>`, `<body>` 태그 정상 닫힘
- [ ] 스크립트 `<script>` 태그 정상 닫힘
- [ ] 모달 오버레이 `modal-overlay` 구조 유지
- [ ] ID 중복 없음 (getElementById 대상 ID들)

#### B. JavaScript 기능 동작
- [ ] `saveFitness()` 함수 존재 및 정상 구조
- [ ] `saveFamily()` 함수 존재 및 정상 구조
- [ ] `renderAll()` 함수 존재 및 정상 구조
- [ ] `loadData()` / `saveData()` Firebase 연동 코드 유지
- [ ] `state` 객체 DEFAULT 정의 유지
- [ ] 새 필드 추가 시 DEFAULT에도 추가됐는지 확인
- [ ] 새 필드 추가 시 loadData 마이그레이션 코드 확인

#### C. 기존 기능 파손 여부
- [ ] 조깅 입력 로직 유지
- [ ] 턱걸이/스쿼트/팔굽혀펴기 날짜 배열 방식 유지 (`pullup_days[]`)
- [ ] 가족여행 횟수 누적 방식 유지 (`travel_months[]`)
- [ ] 교육 분기 기록 방식 유지
- [ ] 주식/연금 수익률 입력 유지
- [ ] 오버뷰 카드 4개 퍼센트 계산 유지

#### D. 데이터 안전성
- [ ] Firebase 설정값(apiKey, appId 등) 변경 없음
- [ ] `DOC.set(state)` / `DOC.get()` 패턴 유지
- [ ] 기존 Firestore 필드명 변경 없음 (하위 호환 깨짐 주의)

#### E. 변경 범위 적절성
- [ ] 요청된 작업 범위 내에서만 수정됨
- [ ] 의도하지 않은 추가 변경 없음

### 3. 보고서 출력

다음 형식으로 반드시 보고:

```
📋 검수 보고서
━━━━━━━━━━━━━━━━━━━━━━
브랜치: <브랜치명>
커밋: <최신 커밋 해시>
검수 시각: <현재 시각>

📌 변경 요약
- <변경된 항목들>

✅ 통과 항목
- ...

⚠️ 이슈 발견  ← 없으면 이 섹션 생략
- [심각도: 높음/중간/낮음] 이슈 설명
  → 재작업 필요: <구체적으로 무엇을 어떻게 수정해야 하는지>

━━━━━━━━━━━━━━━━━━━━━━
🟢 배포 가능  /  🔴 재작업 필요
```

### 심각도 기준
- **높음**: 사이트 접속 불가, 데이터 손실 위험, Firebase 연동 파손
- **중간**: 특정 기능 동작 안 함, 잘못된 계산, 데이터 저장 오류
- **낮음**: UI 어색함, 텍스트 오타, 미세한 스타일 불일치

## 배포 가능 판정 기준
- 높음 이슈 0개 → 🟢 배포 가능 (중간/낮음은 사용자 판단)
- 높음 이슈 1개 이상 → 🔴 재작업 필요

## 배포 가능 시 안내
```
배포하려면 다음을 실행하세요:
  git checkout main && git merge <브랜치명> && git push origin main
또는 auto-merge 워크플로우가 push 시 자동 실행됩니다.
```
